# 墨研论文助手

一个用于论文降重、降 AIGC 与学术润色的网页端项目。当前版本已接入 DeepSeek 后端代理，支持逐段落流式改写与修改前后对比。

## 功能范围

- 原文输入与示例填充
- 改写模式选择：学术规范、自然表达、保守改写
- 改写强度调节
- DeepSeek 多 AI 协作降重
- 判断输入是全篇论文还是片段段落
- 全篇论文按章节/小节生成大纲卡片
- 流式输出改写内容
- 修改前 / 修改后逐段对比
- 段落总结、上下文衔接、图表检测和终审意见展示
- 结果导出
- 合规提示与学术边界说明

## 本地运行

复制环境变量示例：

```bash
cp .env.example .env
```

在 `.env` 中填写：

```text
DEEPSEEK_API_KEY=你的 DeepSeek API Key
DEEPSEEK_MODEL=deepseek-chat
PORT=8787
```

安装依赖：

```bash
npm install
```

启动前端和后端：

```bash
npm run dev
```

构建生产包：

```bash
npm run build
```

预览生产包：

```bash
npm run preview
```

## 当前说明

当前版本使用 `server/index.js` 作为后端代理，浏览器不会直接接触 DeepSeek API Key。

前端请求路径：

```text
POST /api/rewrite-stream
```

后端会将论文按空行切分为段落，然后逐段调用多个 DeepSeek 角色，并通过 `text/event-stream` 将结果推送给前端。

在逐段改写前，后端会先进行结构识别：

- 如果检测到多个章节、小节标题，判定为全篇论文。
- 如果没有明显标题，判定为片段段落。
- 全篇论文会按小节生成大纲卡片，并把每个段落归属到对应小节。
- 片段段落会归入“片段内容”卡片。

## 多 AI 协作角色

- **段落总结与逻辑分析员**
  - 总结每段核心内容。
  - 判断当前段和上下文的逻辑关系。

- **上下文衔接员**
  - 分析上一段、当前段、下一段之间的承接关系。
  - 给出改写时的衔接建议。

- **图表需求检测员**
  - 检测当前段是否需要流程图、对比表、数据表或示意图。
  - 给出图表类型、内容和素材需求。

- **终审员**
  - 检查事实边界、术语边界和降 AIGC 风险。
  - 给最终改写设定审查约束。

- **降重降 AIGC 改写员**
  - 根据前面角色的分析结果输出最终改写文本。
  - 最终改写继续使用流式输出。

## 方案建议

第一版采用逐段落降重是合理的，优点是：

- 单次上下文更短，响应更稳定
- 便于失败重试
- 便于前端展示每段修改前后对比
- 更适合流式输出

更好的进阶方案是“两阶段改写”：

1. 先分析全文，提取研究主题、核心术语、关键变量、表达风格和禁改内容。
2. 再逐段落降重，并把第一步得到的全局约束传给每段改写任务。

这样可以减少逐段改写导致的术语不统一、语气不一致和上下文断裂问题。

后续可接入：

- 论文查重接口
- AIGC 检测接口
- Word / PDF 文档解析
- 用户登录、订单、历史记录
